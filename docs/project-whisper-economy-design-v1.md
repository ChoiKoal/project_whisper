# Part A — 재화 체계 전체 맵

## A-1. 재화 4계층 정의

원본 문서는 "재화"를 단일 개념(기본 에너지)으로만 뒀지만, 실제 시스템(Collect/Fusion/Echo/Whisper)이 요구하는 자원 성격이 서로 달라서 **4계층으로 분리**한다. 계층을 섞으면 "무한이어야 할 것"과 "한정이어야 할 것"이 한 통에 들어가 밸런싱이 불가능해진다.

| 계층 | 이름 | 성격 | 무한/한정 | 주 소모처 |
|---|---|---|---|---|
| ① 소재 | 채집·조합 아이템 (31종) | 재료 | **사실상 무한** (리스폰 + Echo 복제) | Fusion / Build / 배치 |
| ② 에너지 | **정수(Essence)** | 행동 연료 | **소프트 한정** (탐험으로 상시 회복) | Echo 복제 / 일부 행동 |
| ③ Whisper 재화 | **에너지·마력·생명** 3종 | 부여 자원 | **하드 한정** (회차로 점진 해제) | Whisper 부여 |
| ④ (파생) | Balance 지표 | 재화 아님(상태값) | — | Part C |

핵심 원칙: **"발견은 어렵고 재사용은 싸다"**. ①②는 노가다를 없애기 위해 관대하게, ③은 페이싱을 잡기 위해 엄격하게.

## A-2. ① 소재 — 사실상 무한

| 항목 | 정의 |
|---|---|
| 획득처 | 타일/오브젝트 채집(Collect), Fusion 결과물, Echo 복제 |
| 소모처 | Fusion 투입, Build 투입, 세계 배치(디딤돌 등) |
| 상한 | 인벤토리 스택 상한 **999/종** (사실상 무제한, UI 안전장치용) |
| 리스폰 | 일반 채집 타일은 채집 후 **VOID → 실시간 리스폰 아님**. 대신 **한 번 조합한 소재는 Echo로 재생산**(원본 §Echo) → 무한 공급은 리스폰이 아니라 Echo가 담당 |
| 예외 | **세계수 정수(I9)** = 유니크·복제 불가 (Part B-2에서 상세) |

> 설계 포인트: 원본은 "탐험으로 쉽게 채집"이라 했지만 VOID(채집 시 타일이 빈 타일로 변함, GDD 2.1)와 충돌한다. 채집 타일이 무한 리스폰하면 VOID의 "세계를 비운다"는 테마가 죽는다. **해결: 타일 채집은 1회성(VOID 발생), 무한 공급은 Echo가 맡는다.** 채집=유한·서사적, Echo=무한·기계적으로 역할 분리.

## A-3. ③ Whisper 재화 3종 — GDD 2.5 대응 채택

**결정: GDD 2.5의 "Whisper 3종 = 레이어 테마 대응"을 채택한다.**
**근거: 탐험 동기를 재화 획득처로 강제할 수 있고, 원본의 "특정 조건 맵에서만 획득"과 정확히 맞물린다.**

| Whisper | 재화명 | 획득 레이어 | 해금(업적) | 부여 효과 |
|---|---|---|---|---|
| 1차 | **에너지(Energy)** | L3 기계 / L2 과학 | 1차 업적 | 오브젝트 동력화 |
| 2차 | **마력(Mana)** | L4 마법 | 2차 업적 | 마법 특성 부여 |
| 3차 | **생명(Vita)** | L1 자연 / L5 신성 | 3차 업적 | 생명 부여 |

- 원본은 "에너지/마력/생명"을 부여 *종류*로만 정의 → 여기서 각 종류에 **전용 한정 재화**를 1:1 대응시킨다(부여하려면 그 재화가 있어야 함).
- MVP(L1 자연·시작의 숲)에서는 **Vita(생명)만 미세하게 노출** 가능하나, 원칙적으로 **MVP는 Whisper 재화 전면 제외**(Part D 참조).

## A-4. 미결 핵심 — "한정 vs 무한"

**결정: 3-tier 혼합. 소재=무한(Echo 경유), 정수=소프트 한정, Whisper 재화=하드 한정.**
**근거: 단일 정책(전부 무한 or 전부 한정)은 둘 다 실패한다. 전부 무한이면 Whisper의 "페이싱 장치"(원본 §Whisper, GDD 2.5) 기능이 사라지고, 전부 한정이면 Doodle God식 발견·재사용 놀이가 노가다가 된다. GDD 2.1 제안("일반 무한 + Whisper만 한정")을 채택하되, 그 사이에 정수(②)를 끼워 Echo에 마찰(cost)을 줘서 "복제가 완전 공짜"가 되는 것을 막는다.**

### 보유 곡선 (예상, 1회차 기준)

| 구간 | 소재(종류 발견) | 정수 보유 | Whisper 재화 |
|---|---|---|---|
| 초반(L1 튜토리얼) | 9→22종 | 20~40 (여유) | 0 |
| 중반(L2~L4 탐험) | +60~80종/레이어 | 30~60 진동 | 레이어당 5~15 획득, 즉시 대부분 소모 |
| 후반(L5+제0세계) | 도감 대부분 | 40~80 | 총 보유 상한 근접, 엔딩 배분 압박 |

# Part B — 수지 설계 (수치)

## B-1. MVP(시작의 숲) 구체 수치

### 채집 획득량

| 대상 | 1회 채집 획득량 | 근거 |
|---|---|---|
| 일반 타일(흙/풀/바위/돌/물 등) | **3개** | 2-조합에 최소 2개 필요 + 실패 여유 1. 스택 부담 없이 "충분하다" 감각 |
| 오브젝트(꽃/나무) | **2개** | 희소감 부여 |
| 세계수 정수(I9) | **1개(유니크·영구 소모)** | 서사적 유일성 |

### Echo 복제 비용 (정수 소모)

**결정: 티어별 계단 비용. 키 아이템은 복제 불가.**
**근거: 발견 난이도에 비례한 재생산 비용 → "쉬운 건 싸게, 귀한 건 비싸게". 완전 공짜면 정수 계층(②)이 무의미해지고, 너무 비싸면 노가다 제거라는 Echo의 존재 이유가 죽는다.**

| 티어 | 예시 | Echo 비용 | 비고 |
|---|---|---|---|
| T0 기본 채집 | 흙/풀/물/바위/돌 | **1 정수** | 사실상 채집이 더 쌈 → Echo는 편의 |
| T1 파생(2조합) | 진흙/모래/씨앗/디딤돌 | **5 정수** | 요청서 예시값 채택 |
| T2 파생(체인/3조합) | 목재/벽돌/새싹→묘목 | **12 정수** | |
| T3 상위(Whisper 재료) | 정령꽃/생명수 | **25 정수** | |
| 키 아이템 | **세계수 정수(I9), 어린 세계수(D22)** | **복제 불가** | Part B-2 softlock 핵심 |

### 정수 획득 수단

**제안 3안 → 추천 1개:**

| 안 | 방식 | 장점 | 단점 |
|---|---|---|---|
| (가) 채집 부산물 | 채집 시 확률적 정수 드랍 | 탐험 = 자연 수급 | VOID 미발생 채집엔 안 붙음 |
| (나) **VOID 생성 시 드랍** | 타일이 VOID로 변할 때 **정수 +2** 확정 | **테마 직결**: "세계를 비운 대가로 창조 연료를 얻는다" | 채집 안 하면 고갈 |
| (다) 실패 조합 환급 | 실패 시 정수 소량 | 실패 스트레스↓ | 브루트포스 조장(Part B-3와 충돌) |

**추천: (나) VOID 생성 시 정수 +2 확정 드랍.**
**근거: "채집=소비, 창조=회복"(레시피 §1)이라는 코어 테마를 재화 수급에 그대로 새긴다. 채집(VOID 발생)이 곧 정수 공급이므로 탐험량과 Echo 사용량이 자동 연동된다. (다)는 Part B-3의 힌트 브루트포스 문제와 정면충돌하므로 기각.**

## B-2. 퍼즐 게이트 G1~G4 — softlock 불가능 증명

### 게이트별 최소 재료 vs 맵 총량

| 게이트 | 필요 아이템 | 최소 소요 | 맵 확보 가능량 | 여유 |
|---|---|---|---|---|
| G1 시냇가 | 디딤돌(D14 = 바위+돌) ×**3**(물폭 3타일 가정) | 바위 3 + 돌 3 | 바위·돌 각 T0 채집 **무제한 Echo 가능** | ✅ 무한 |
| G2 마른 덤불 | 물 아이템(I7) ×1 | 물 1 | 물 타일 채집 + Echo | ✅ 무한 |
| G3 세계수 입구 | (시간대 조건, 재료 0) | — | — | ✅ 재료 무관 |
| G4 클리어 | 세계수 정수(I9) ×1 → 생명수 → 빛나는 새싹 → 어린 세계수(D22) | I9 **1개(유니크)** | I9 = 맵에 **1개만 존재, 복제 불가** | ⚠ 유일 자원 |

### softlock 위험 지점 = G4의 유니크 소모

**위험 시나리오:** 플레이어가 세계수 정수(I9)를 채집 → 생명수(D19)로 조합 → 그런데 그 생명수를 **G4 아닌 다른 조합(정령꽃 D21 등)에 소진**해버리면, I9는 복제 불가·재획득 불가 → **영구 softlock.**

**해결 = Echo 정책 + 소모 시점 설계:**

**결정: "키 파생물 조건부 Echo 해제". I9 자체는 복제 불가지만, I9로 만든 첫 파생(생명수 D19)은 G4 클리어 전까지 Echo 가능하게 연다.**
**근거: 유니크의 서사(정수는 하나뿐)를 지키면서, 플레이어 실수로 인한 softlock을 차단한다. "정수는 유일하나 그 생명은 복제된다" — 테마적으로도 어긋나지 않음.**

구체 규칙:
1. **I9(세계수 정수)**: 유니크, 복제 불가, 채집 1회.
2. I9 → 생명수(D19) 조합 시, **D19에 "복제 허용" 플래그 부여** (Echo 25정수). 즉 D19가 있으면 언제든 재생산 가능 → I9를 실수로 딴 데 써도 D19 라인은 유지.
3. **최소 안전 경로 보장**: I9로 D19를 만들면, D19에서 D20(빛나는 새싹)→D22(어린 세계수)까지 전부 무한 Echo 가능. 정령꽃(D21) 같은 사이드 조합에 D19를 다 써도 D19를 Echo로 다시 뽑으면 됨.
4. **최후 안전장치**: 만약 I9를 채집 전 상태에서 (버그 등으로) 잃으면, G3 세계수 구역 재진입 시 정수 리젠(회차 개념 없는 MVP 한정 예외).

**증명 결론:** G1~G3은 T0 채집물(무한 Echo)로만 해결 → 절대 막힘 없음. G4는 I9 유니크 소모가 유일 위험이나, **첫 파생(D19)에 복제 플래그를 붙이는 순간 하류 전체가 무한 공급**이 되어 softlock 불가능. **QED.**

## B-3. 힌트 게이지 경제 — 브루트포스 봉쇄

### 문제 정식화

- 31종 아이템, 2-조합 전수 = C(31,2) = **465가지**.
- 실패 비용 0(재료 미소모) + 5회 실패마다 힌트 → **무지성 전수조사가 최적전략**이 됨. 발견의 재미(코어) 붕괴.

### 해법 4후보

| 후보 | 메커니즘 | 브루트포스 억제 | 스트레스 |
|---|---|---|---|
| (가) 실패 쿨다운 | 실패 후 N초 대기 | 강 | **높음(짜증)** |
| (나) **중복 실패 미적립** | *같은 조합* 재실패는 게이지 +0 | 중~강 | 낮음 |
| (다) 조합 후보 시각 피드백 | 반응할 것 같은 재료 발광 | 강 | 낮음(오히려 재미↑) |
| (라) 정수 소액 과금 | 조합 시도당 정수 -1 | 중 | 중 |

**추천: (나) + (다) 병행.**
**근거:**
- **(나) 중복 실패 미적립**: 브루트포스의 본질은 "이미 틀린 조합을 게이지 벌려고 반복"하는 것. 같은 pair 재실패에 게이지를 안 주면, 게이지를 채우려면 **최소 5가지 서로 다른 조합**을 새로 시도해야 함 → 무의미 반복이 봉쇄됨. 진짜 탐색만 보상.
- **(다) 시각 피드백**: 두 재료를 솥에 올렸을 때 "반응 가능성 있음"을 은은한 발광으로 알려주면, 465 전수가 아니라 **의미 있는 후보만 시도**하게 유도. 발견을 '노동'에서 '추리'로.
- (가)는 스트레스로 기각, (라)는 정수 계층과 얽혀 초반 밸런스를 흔들어 보류.

추가 수치: 게이지는 **서로 다른 실패 조합 5종 누적 → 힌트 1회**. (다)의 발광은 **정답 pair의 60% + 오답 근접 pair 20%**를 섞어 확정 스포일러가 되지 않게.

## B-4. Whisper 재화(정식 버전) 수지

| 회차 | Energy 상한 | Mana 상한 | Vita 상한 | 해금 |

| 1회차 | 10 | 6 | 4 | 기본 |
| 2회차 | 16 | 10 | 7 | +엔딩 분기 관찰 가능 |
| 3회차+ | 24 | 16 | 12 | 전 엔딩 도달 가능 재화량 |

**근거: 1회차엔 "한 축밖에 못 밀어붙이는" 부족함으로 첫 엔딩을 강제 편향시키고(멸망 재현 학습), 회차가 쌓일수록 균형 잡힌 창조(진엔딩)가 가능해지도록 재화 총량이 엔딩 요구치를 서서히 따라잡게 설계.** (Part C 임계값과 연동)

# Part C — Balance 시스템 구체화 (정식, GDD 2.6 B안)

**결정: 정식 버전은 B안(원소 조화 지표)을 채택. MVP는 제외(C안).**
**근거: B안이 "선배들의 세계는 왜 멸망했는가"(조화 독주=정체 L1, 경쟁=파괴 L2 …)라는 서사와 시스템을 하나로 묶는다. GDD 2.6 추천과 일치.**

## C-1. 4축 게이지

원본 4채집(물/불/흙/공기)이 아니라, **레이어 테마 4축**으로 잡는다(Whisper 재화와 정합):

| 축 | 의미 | 기여 오브젝트 예 |
|---|---|---|
| 자연(Nature) | 생장·유기 | 나무/이끼/꽃/정령꽃 |
| 에너지(Energy) | 동력·기계 | Energy 부여물, 기계 구조 |
| 마력(Mana) | 변형·마법 | Mana 부여물 |
| 생명(Vita) | 의지·창조 | Vita 부여물, 생물 |

- 제0세계에 배치/Build/Whisper한 각 오브젝트가 대응 축에 **가중치 기여**(예: 정령꽃 Nature+2, Energy 동력화 오브젝트 Energy+3).
- 축 값은 실시간 누적, 제0세계 상태 화면에 방사형(레이더) UI로 표시.

## C-2. 한 축 독주 페널티 (멸망 재현)

- 한 축이 전체의 **일정 비율 이상 독점**하면 그 레이어의 멸망 연출을 제0세계에 재현:
  - **Nature 독주 → 정체(L1 재현)**: 세계가 "시간이 멈춘 듯" 채도 저하·성장 정지 연출.
  - **Energy 독주 → 고갈(L3 재현)**: 동력 오브젝트 점멸·정지.
  - **Mana 독주 → 왜곡(L4 재현)**: 타일 뒤틀림(정화의 계곡 이상반응 확장).
  - **Vita 독주 → (L5)**: 피조물 폭주/이탈.
- 페널티는 하드 실패가 아니라 **엔딩 분기 신호** — 그 축 특화 엔딩으로 유도.

## C-3. 균형도 수식 (측정 가능)

4축 값을 `n_i` (i∈{Nature,Energy,Mana,Vita}), 총합 `N = Σn_i`, 비율 `p_i = n_i / N`.

**정규화 섀넌 엔트로피 기반 균형도 B ∈ [0,1]:**

```
B = − Σ p_i · log(p_i) / log(4)
```

- 완전 균형(각 25%) → B = 1.0 (진엔딩 후보)
- 완전 독주(한 축 100%) → B = 0.0 (해당 축 멸망 엔딩)
- log 밑은 4(축 개수) → 축 수 바뀌어도 [0,1] 정규화 유지.

### 엔딩 분기 임계값

| B 범위 | 지배 축 | 엔딩 |
|---|---|---|
| B ≥ 0.90 | 없음 | **진엔딩(조화의 세계)** — 회차 누적 재화 필요 |
| 0.55 ≤ B < 0.90 | 약한 편향 | 편향 엔딩(해당 축 색채 세계) |
| B < 0.55 | 강한 독주 | **멸망 재현 엔딩**(선배 레이어 반복) — 서사적 "실패지만 교훈" |

- 1회차는 재화 상한(Part B-4)상 B ≥ 0.90 달성 불가 → 자연스럽게 첫 회차는 편향/멸망 엔딩으로 귀결 → 회차 반복 동기.

# Part D — 구현 우선순위

## D-1. MVP 포함/제외 결정

**결정: MVP는 재화 시스템 전면 제외. 단 "데이터 훅"만 심는다(C안 확장).**
**근거: 현 MVP 스코프(GDD §6)는 채집→Fusion→발견 검증이 목표이며, 정수·Echo·Whisper 없이도 완결된다. 레시피 문서의 G1~G4도 재화 없이 성립(디딤돌 배치·물 사용·시간·조합 체인). 재화를 지금 넣으면 검증 대상(발견의 재미)이 흐려지고 밸런싱 변수만 폭증.**

단, **완전 제외가 아니라 "훅 예약"**:

| 항목 | MVP | 훅(데이터 구조만) | 정식 |
|---|---|---|---|
| 소재/인벤토리 | ✅ 구현 | — | — |
| Fusion/도감 | ✅ 구현 | — | — |
| 힌트(중복 미적립+시각 피드백) | ✅ **구현**(B-3) | — | — |
| 정수(Essence) | ❌ | 아이템에 `echo_cost` 필드·정수 카운터 스텁 | ✅ 활성화 |
| Echo | ❌ (MVP는 채집 3개로 충분) | 복제 함수 스텁 | ✅ |
| Whisper 3종 | ❌ | 아이템 `whisper_type` enum 예약 | ✅ |
| Balance 4축 | ❌ | 오브젝트에 `axis_weight` 메타 예약 | ✅ B안 |

- **왜 훅은 남기나:** 정식에서 아이템 31종 + 향후 레이어 아이템에 Echo 비용·축 가중치를 **소급 부여하면 데이터 마이그레이션 비용이 큼.** MVP 아이템 정의 스키마에 빈 필드만 예약해두면 정식 전환 시 값만 채우면 됨.

## D-2. 왜 정수+Echo도 MVP에서 빼는가

- 후보였음: "정수+Echo만 MVP에 넣자". 그러나 채집 획득량을 **3개**(B-1)로 잡으면 G1~G4 클리어에 재생산이 불필요 → Echo의 존재 이유(노가다 제거)가 MVP 볼륨(레시피 22종)에서는 발생하지 않음. **작은 맵에선 Echo가 무의미하므로 뺀다.** Echo는 재료 재사용이 잦은 제0세계 구축 모드가 들어올 때 켜는 게 옳다.

# L0 허브 확장 패스 — 미니 설계 (v1.10.0, 태스크 #254)

## 목표
제0세계(home island) L0 허브를 **21×17 → 31×25**로 확장. 승인된 포탈 아치 구도를
**셀 좌표 그대로 보존**하고 주변으로만 확장해, 허브다운 밀도·장소감·세계층 방향성을
부여한다. (KOAL 승인 구도 불변 — 훼손 시 반려)

## 절대 불변 (v1.4.2 복원 구도)
staggered(STACKED) 투영: `local = ((c + 0.5·(r&1))·128, r·32)`. 월드 좌표는 **절대 셀
인덱스**의 선형함수이므로, 코어 셀의 (c,r) 인덱스를 **그대로** 두면 `cell_center_world`
반환값이 픽셀 단위로 동일 → 하네스 아치 어서션 전부 불변 통과.

확장 규칙: **기존 콘텐츠 셀의 (c,r) 인덱스를 이동시키지 않는다.**
- 우측(c 증가)·하단(r 증가) 신규 셀 추가 → 인덱스 무이동.
- 상단/좌측은 **기존 V 패딩 셀(고정 인덱스)** 을 V→지면(D)으로 승격해 실루엣만 확장.
  V→지면은 세이브 클램프에 무해(지면→V는 절대 없음).

## 확장 레이아웃 초안 (31×25)
- 폭 21→31 (+10 col, 우측 확장), 높이 17→25 (+8 row, 하단 확장).
- **섬 실루엣 비대칭화**: 마름모 코어를 유지하되 가장자리를 불규칙 톱니로. 남동/남서로
  넓게 흐르는 하단 앞마당(제단·솥·게이트 기능구역 분리용 여백), 좌측은 기존 V패딩을
  지면화해 자연스러운 곶(promontory), 우측은 관측석 너머로 뻗는 관측 테라스.
- **기능 구역 분리 (여백)**:
  - 상단 밴드(r 2–6): 포탈 아치 제단 구역 — 각 게이트 앞 **층별 상징 데코 소품**.
  - 중앙(r 7–11): 다이스 제단 광장 — 넓은 빈 바닥(§㉚ 목적 있는 휴지), 빛 웅덩이·사인.
  - 좌하(r 11–16): 솥 작업 구역 — 솥 주변 비석·잔해.
  - 하단 앞마당(r 16–24): 부유 파편이 짙어지는 가장자리, 엣지 그림자 보강.
- **세계층 방향성 데코** (각 포탈 앞 apron 바로 위, 진입 apron 셀 (col,row+2)은 보행
  가능 유지 → 데코는 apron 옆/뒤에만):
  - P1 자연 → **잎** 모티프: 이끼 돌·새싹 (심볼 `p`).
  - P2 과학 → **데이터** 모티프: 발광 룬 결정 (심볼 `q`).
  - P3 기계 → **태엽** 모티프: 멈춘 톱니 비석 (심볼 `k`).
  - P4 마법 → **서고** 모티프: 부유하는 룬 서판 (심볼 `b`).
  - P5 신성 → **종** 모티프: 작은 석종·향로 (심볼 `n`).
- **프로시저럴 밀도 보강 (§㉙ 변주 / §㉚ 목적 있는 바닥 휴지 / §㉛ 목표물 대비)**:
  - 빛 웅덩이(`o`) 스캐터 — 게이트/제단 시선 유도, 저대비.
  - 비석·잔해(`x`) — 솥·가장자리에 낮은 밀도, 실루엣 변주(art_variants).
  - 부유 파편·하단 그림자 밀도는 floating_shard(map_loader)가 자동 처리 →
    슬랩이 커지면 aprons/underside/debris가 확장분까지 자동 확대.
  - §㉛: 데코는 전부 저채도·저높이라 게이트(목표물) 실루엣 대비를 흐리지 않음.

# Part A — Layer 2 구역1 「꺼진 관문 기지」 40×40 타일 설계

## A-1. 컨셉

자원 전쟁으로 폐허가 된 문명의 **관문 기지(gate base)**. 중앙 관제탑을 정점으로 계단식 플랫폼이 솟아 있고, 그 아래를 오염된 냉각수 협곡이 가른다. 모든 기계는 꺼져 있고, 스크린은 깜빡이다 죽고, 에너지는 갈라진 도관에서 시안색으로 새어 나온다. 플레이어는 **전력을 한 노드씩 되살려** 관제탑 코어에 도달한다.

무드: 남색 base(#1a2438) + 시안 발광 액센트(#4ad9c8) + 금속 회색 램프. Layer 1의 "멈춘 세계"가 **자연의 정적**이었다면, Layer 2는 **기계의 정적** — 소리 없이 깜빡이는 죽은 불빛.

## A-6. 채집원 충분성 — softlock 불가 논증

각 게이트 체인이 요구하는 원소가, 그 게이트 **앞 지대에서** 충분히 확보 가능함을 증명(뒤 지대 재료로 앞 게이트를 여는 순환 의존이 없음).

|---|---|---|---|---|---|
| G1 브리지 | 전지 ×1 | 회로(J4)+기름(J5)+고철(J1) | 남 광장/황무지(row 29–38) | ✅ 앞 | s 부품상자 4+, R/m 고철 8+, oil_leak 2 |

| G2 퓨즈 | 퓨즈 ×1 | 전선(J2)+회로(J4)+유리(J3) | 네온/재 지대 + 중앙 플랫폼 s | ✅ 앞(G3 통과 후 플랫폼) | s(플랫폼) + 앞 지대 잔여 |

- **핵심 불변식**: *N번째 게이트의 재료는 N-1번째 게이트까지 열린 영역에서 전량 확보 가능.* 위 표의 "게이트 앞?" 열이 모두 ✅ → 어떤 게이트도 그 뒤 지대의 재료를 요구하지 않음 → **데드락/softlock 불가**.
- **비소비 소지형(G3) 특례**: 네온 랜턴은 소지형 게이트라 **소비되지 않으므로**, G3 통과 후에도 계속 소지 → 이후 왕복(재료 채집 위해 남쪽 복귀) 시 재차 통과 가능. 랜턴을 실수로 버려도 네온 결정(J6)이 리스폰되므로 재제작 가능. (items.json에서 네온 랜턴 = `key_item:true`, 인벤 고정 권장 → 오배치 방지.)
- **원소 리스폰**: J1~J7 채집원은 Layer 1 정책 계승(채집 후 실시간 ~하루 리스폰). 재(A)·네온(N채집)은 타일/군락 채집이나, Layer 2는 "정화=전력 복구"가 테마이므로 **채집=영구 소멸 딜레마를 적용하지 않고 리스폰** — VOID 딜레마는 Layer 1 고유. (근거: 세계관상 자원은 '소진'이 아니라 '재분배' 대상. Owner 확정 축이 전력이지 자원 고갈이 아님.)

## B-3. 무결성 자체 검증

> 아래 3항은 실제 데이터(`game/data/recipes.json` 62종)와 스크립트 대조로 확인했다(§Part B 말미 '검증 실행 결과'). 결정마다 왜 안전한지 논증한다.

### (1) 페어 중복 없음 (기존 62종 recipes.json 대조)

- **원리**: 조합은 `frozenset(inputs)` 무순 2원소로 매칭. 페어검사 대상 41쌍(L2-R08 = 조각+에너지 제외) 중 **J-원소를 하나라도 포함한 30쌍은 구조적으로 충돌 불가** — 기존 62종은 J-원소를 쓰지 않기 때문. 위험은 **순수 I/D 재료 쌍 11개**(예: L2-R28 강철 도끼 = D57+D70, L2-R03 전지 = D62+D63)인데, D62~ 산출은 전부 신규(Layer 2)라 기존 어디에도 없어 → 전부 안전(스크립트 [3] 로그로 개별 확인).
- **자기 충돌**(신규 쌍 내부 중복)도 스크립트로 검사 → 초안에서 4건 검출 후 재료 교체로 해소, 최종 중복 0(§검증 실행 결과).
- **왜 이 방식**: 프리픽스 분리(J/D62~)가 곧 네임스페이스 격리라, "설계 시점에 이미 대부분 안전"이 보장되고 스크립트는 잔여 I/D 쌍만 확인하면 된다.

### (2) G4 체인 softlock 불가 (Part A §A-6 소스 개수 표와 연결)

- 게이트별 필요 최종물과 소비 원소가 **그 게이트 앞 지대에서 전량 확보**됨은 §A-6 표(모든 '게이트 앞?' = ✅)가 이미 증명. Part B는 그 원소들을 **구체 레시피로 전개**했을 뿐, 새로운 뒤-지대 의존을 추가하지 않았다:
  - D64 전지 ← (J1,J2,J4,J5) : 전부 **남 광장/황무지**(브리지 앞). §A-6 소스: s 4+, R/m 8+, oil_leak 2. **✅**

  - D66 퓨즈 ← (J1,J2,J3) : 브리지 북~플랫폼(G2 앞). **✅**

## C-3. 신규 시스템 2개 (기존 패턴 재사용)

> **원칙**: 게이트 시그널 패턴을 최대 재활용. Layer 1이 `stepping_stone_placed`/`item_used_on_object`/`world_tree_planted` 3종으로 4게이트를 처리했듯, Layer 2도 **기존 시그널 + 신규 2개**로 처리.

### (1) 전력 노드 (power_node) — 기존 게이트 시그널 패턴 재사용

- **대응물**: Layer 1 `stepping_stone_placed(cell)` ↔ Layer 2 **`power_node_energized(node_id)`**. 배전반 K에 열쇠 아이템(전지 D64 / 파워 코어 D69) 장착 = 노드 급전.
- **물리는 곳**: `GameState`에 시그널 1개 추가(`signal power_node_energized(node_id: String)`) + `powered_nodes: Dictionary`(세이브 대상, `home_session`의 `portal_states` 저장 패턴 복제). 배전반 오브젝트 씬 = Layer 1 배치 슬롯(디딤돌 K)의 **장착형 변형** — placement 프레임워크(`placeable_on`/`place_slot`)를 "소비 배치" 대신 "소켓 장착"으로 재사용.
- **게이트 소비**: G1(브리지)·G4(관제탑)가 `power_node_energized`를 listen. G1은 노드 급전 시 브리지 B 셀 충돌 해제(walkable 스왑, Layer 1 디딤돌의 물→흙 스왑과 동형). G4는 관제탑 노드 급전 시 클리어 컷신 훅(Layer 1 `world_tree_planted` 훅 자리 그대로).
- **왜 새 시그널 최소화**: Layer 1의 배치/사용/심기 3종 중 '배치'만 '장착'으로 의미 확장하면 되므로 시그널 이름 1개만 신설.

### (2) 소지형 게이트 (held-item gate) — 인벤토리 보유 체크 Area2D

- **대응물**: Layer 1 G3(시간형, `game_phase`가 밤이면 N 통행) ↔ Layer 2 G3(**소지형**, 인벤에 네온 랜턴 D65 보유 시 정전 병목 통행).
- **물리는 곳**: N 병목 셀(17,14)(18,14)(17,15)(18,15)에 **Area2D**(Layer 1 G3 밤길 Area2D 씬 복제). `_process`(또는 `body_entered`)에서 `Inventory.has_item("D65")` 폴링 → true면 충돌 해제 + 주변 조명 반경 생성(local light decal), false면 국소 암전(비네트 극대화) + 충돌 유지. Layer 1 시간형이 `game_phase` 조건을 보던 자리를 **인벤 보유 조건**으로 치환 = 로직 골격 동일, 조건식만 교체.
- **QuestManager 연동**: 신규 시그널 불필요. 기존 `player_entered_area`(랜턴 소지 상태로 병목 진입 시 발화) 또는 신규 `held_item_gate_passed(gate_id)` 중 택1 — 전자 재사용 권장(QuestManager `target` 필터만 추가).
- **왜 폴링 허용**: 인벤 변경 시그널이 없다면 Area2D 내부에 있는 동안만 매 프레임 `has_item` 체크(가벼움) — Layer 1 G3의 `_process` game_time 폴링과 동일 비용.

## C-4. 구현 순서

> Layer 1 자산·로더·게이트·퀘스트·포탈 파이프라인을 **순서대로 재사용**하며 Layer 2를 얹는다.

1. **타일** — §C-1 리컬러 셰이더 + 데칼 6종. `whisper_tileset.tres`에 Layer 2 source 추가(기존 source-id 규약 유지, VOID=0).

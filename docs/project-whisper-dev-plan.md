## 0. 목표와 완료 정의

**MVP 목표: "1구역(시작의 숲)에서 채집→조합→발견 루프가 재미있는가"를 mac/Windows 빌드로 검증.**

완료 정의(DoD):
- macOS(.dmg 또는 .app)와 Windows(.exe) 빌드가 실행됨
- 시작의 숲 맵에서 이동/채집/조합/도감/세이브가 전부 동작
- 레시피 20종 이상, 아이템 9종 채집 가능
- 크래시 없이 30분 연속 플레이 가능

## 1. 기술 기준 (모든 태스크 공통)

- **Godot 4.x (최신 stable), GDScript**, 렌더러: `gl_compatibility` (구형 mac 대응)
- 프로젝트 설정: viewport 1920×1080, stretch `canvas_items`, `snap_2d_transforms_to_pixel` on
- TileMapLayer: Isometric / Diamond Down / 타일 128×64 / Y-sort on
- 데이터는 코드와 분리: 아이템·레시피·오브젝트 정의는 **JSON 또는 Godot Resource(.tres)** — 기획 수정이 코드 수정이 되지 않게
- ID 체계는 기획 문서 그대로 (T0~T5B, O0~O6, I1~I9)
- git 저장소, 태스크 = 브랜치 = PR 단위. 아트 플레이스홀더(단색 도트)로 개발 선행, 아트 교체는 리소스 스왑만으로 가능하게

## 2. 태스크 분해 (서브에이전트 카드)

각 카드는 독립 실행 가능하도록 입력/산출/인수조건 명시. **P = 선행 태스크.**

### M0. 프로젝트 셋업 `[P: 없음]`
- Godot 프로젝트 생성, 폴더 구조(`/scenes /scripts /data /assets /ui`), git init
- export preset: macOS + Windows Desktop, 실제 빌드 1회 성공
- 인수조건: 빈 씬이 양 플랫폼에서 실행되는 빌드 산출물

### M1. 아이소메트릭 코어 `[P: M0]`
- TileMapLayer 셋업(128×64 diamond), 플레이스홀더 타일 8종(T0~T5B, 단색)
- 플레이어(플레이스홀더 스프라이트) 4방향 이동, 충돌(물 타일 진입 불가, 진흙 감속), 카메라 팔로우
- Y-sort로 오브젝트 앞뒤 정렬 검증 (나무 뒤로 돌아가기)
- 인수조건: 20×20 테스트 맵에서 이동·충돌·정렬 정상

### M2. 채집 시스템 `[P: M1]`
- 상호작용: 인접 타일/오브젝트 하이라이트 + 채집 키
- 타일 채집 → T0(VOID)로 치환, 오브젝트 채집 → 제거(세계수 O0은 1회 채집 후 잔존 예외)
- 인벤토리 데이터 모델 + UI(그리드, 스택), 아이템 정의 `items.json` (I1~I9)
- 채집 경로 매핑은 기획 문서의 표 그대로 (예: I2 풀 ← T2A/B/C/D, O5)
- 인수조건: 9종 아이템 전부 채집·소지 가능, VOID 치환 동작

### M3. Fusion + 도감 `[P: M2]`
- 솥단지 오브젝트 + 조합 UI (슬롯 2~3개 + 조합 버튼)
- 레시피 시스템: `recipes.json` (재료 조합 → 결과, 순서 무관 매칭)
- 실패 조합: 재료 미소모, "반응 없음" 피드백 + 힌트 게이지 누적
- 도감(발견 기록) UI: 발견한 레시피/아이템 열람, 미발견은 실루엣
- 인수조건: 레시피 20종 데이터 투입 시 코드 수정 없이 동작, 도감 기록 유지

### M4. 시작의 숲 맵 구현 `[P: M2, 아트 Batch1 병행]`
- 기획 문서의 맵 스케치 기반: 연못 시작점 → 시냇가 → 풀 언덕 → 세계수 구역
- 오브젝트 배치(O0~O6 변형 포함), 세계수 구역은 저녁~새벽만 진입 가능
- 낮/밤 사이클: 게임 내 시간 + CanvasModulate 틴트 (아트 가이드 §3)
- 인수조건: 스타트→세계수까지 자연스러운 동선, 시간대 제한 동작

### M5. 세이브/로드 `[P: M3, M4]`
- 저장 대상: 인벤토리, 도감, 맵 상태(VOID/채집된 오브젝트), 게임 시간, 플레이어 위치
- 슬롯 1개 + 자동 저장(종료 시), JSON 직렬화, `user://` 경로
- 인수조건: 종료 후 재시작 시 상태 완전 복원

### M6. 폴리시 & 패키징 `[P: 전체]`
- 아트 최종 교체 확인, SFX 플레이스홀더(채집/조합/UI), 타이틀 화면(새 게임/이어하기/종료)
- mac/Win 빌드, 30분 플레이 테스트, 버그 트리아지
- 인수조건: §0 DoD 전체 충족

## 3. 레시피 초안 (MVP 20종 — 기획 확정 필요)

1구역 채집물 9종(흙/풀/진흙/나무/물/꽃/바위/돌/세계수) + 기본 원소 개념으로 구성.
문서에 이미 있는 것: `물+돌=진흙`(→ 기획 문서와 T4 채집의 정합성 확인 필요), `잎사귀+바람=씨앗`.

Tier 1 (원소끼리): 물+흙=진흙 / 풀+물=이끼 / 돌+돌=바위 / 나무+돌=목재도구 / 꽃+물=꽃즙 / 흙+풀=씨앗 / 나무+나무=장작 ...
Tier 2: 씨앗+물=새싹 / 새싹+시간(?)=묘목 / 꽃즙+이끼=물감 / 진흙+장작=벽돌(가마 필요?) ...
Tier 3 (세계수 관련): 세계수+물=생명수 / 생명수+씨앗=빛나는 새싹 ...

→ **전체 레시피 트리는 별도 태스크로 분리** (기획 태스크 R1): 20종 확정 + 힌트 문구 + 도감 플레이버 텍스트. 내가 초안 만들고 KOAL 컨펌 받는 흐름 제안.

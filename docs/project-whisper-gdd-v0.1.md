## 0. 한 줄 컨셉

**"멸망해가는 세계들을 탐험하며 원소를 모으고, 조합(Fusion)과 속삭임(Whisper)으로 나만의 세계를 창조하는 아이소메트릭 도트 샌드박스 어드벤처."**

- 장르: 조합·발견(Alchemy / Doodle God 계열) × 탐험 × 세계 구축(월드 빌딩)
- 뷰: 아이소메트릭(30–45°), 도트 그래픽 (AI 레퍼런스 → 픽셀화 파이프라인 확정됨)
- 플랫폼: **macOS / Windows 패키지 게임** (스탠드얼론, 오프라인)
- 세션 구조: 싱글플레이, 반복 플레이(Loop) 기반 멀티 엔딩

### 차별점 (Doodle God와 뭐가 다른가)
Doodle God류는 "조합 그 자체"가 게임의 전부라 금방 소모됨. Whisper는 조합을 **탐험(재료를 어디서 얻는가)**과 **창조(조합한 걸 어디에 쓰는가)** 사이에 끼워 넣어 루프를 만든다:

```
탐험/채집 (각 레이어) → 조합 (Fusion/Build) → 부여 (Whisper) → 세계 구축 (제0세계) → 새 지역/기능 해금 → 다시 탐험
```

이 루프가 코어. 조합은 목적이 아니라 **세계를 만들기 위한 수단**이다.

## 1. 세계관 요약 (확정된 부분)

- 세계는 여러 개의 "레이어"로 구성. 각 레이어는 멸망 진행 중이며, 컷신으로 서서히 드러남.
- **주인공**: 갓 태어난 컨스트럭터(세계 설계자). 선배들이 만든 세계는 이미 멸망해가고 있음. 왜 멸망하는지 탐험으로 밝혀내고 자신의 세계를 창조한다.

| # | 레이어 | 테마 | 멸망 원인 | NPC |
|---|---|---|---|---|
| 0 | Layer of Possibility | 플레이어의 세계 (구축 공간) | — | [결정 필요] |
| 1 | Layer of Nature | 조화 / 시간이 멈춘 자연 | 완벽한 조화 → 정체 → 시들어감 | 엘프, 수인 |
| 2 | Layer of Science | 과학 / 우주 | 자원 전쟁으로 폐허 | 우주인, 외계인 |
| 3 | Layer of Steampunk | 기계 | 에너지 고갈 | 로봇 |
| 4 | Layer of Magic | 마법 | 금기를 건드림 (봉인 해제) | 현자, 용사 |
| 5 | Layer of Divinity | 신성 | 피조물의 믿음 상실 | 신, 천사 |

**내러티브 축 제안**: 각 레이어의 멸망 원인이 곧 "세계 설계의 교훈"이 됨 — 조화만 있으면 정체하고(1), 경쟁만 있으면 파괴되고(2), 동력은 유한하고(3), 힘에는 금기가 있고(4), 창조자는 피조물과의 관계로 유지된다(5). 플레이어가 제0세계를 만들 때 이 교훈들이 엔딩 분기 조건이 되면 멀티 엔딩과 자연스럽게 연결됨.

## 2. 코어 시스템

### 2.1 Collect (채집)
- 레이어 내 타일/오브젝트에서 원소·아이템 채집.
- 기본 4원소: **물 / 불 / 흙 / 공기**
- 타일 채집 시 해당 타일이 **VOID(빈 타일)**로 변함 → "세계에서 재료를 뽑아간다"는 감각 + 멸망 테마와 연결. 채집이 곧 그 세계를 더 비워가는 행위라는 딜레마를 후반 내러티브 훅으로 쓸 수 있음.
- [결정 필요] 재화(에너지)를 한정으로 할지 무한 채집으로 할지 → **제안: 일반 재료는 리스폰(무한), Whisper 재화만 한정.** 시스템 문서의 "Whisper 재화 제한의 점진적 해제"(Loop 해금)와 맞물림.

### 2.2 Fusion (융합) — 게임의 심장
- 채집한 요소를 **솥단지**에 넣고 조합 → 관계성에 따라 새 오브젝트 생성.
- Doodle God식 발견의 재미: 레시피는 힌트만 주고 직접 발견하게.
- **레시피 설계 원칙 (제안)**:
  - Tier 0: 기본 4원소 + 레이어 고유 채집물 (1구역 기준 9종: 흙/풀/진흙/나무/물/꽃/바위/돌/세계수)
  - Tier 1: 2원소 조합 (예: 물+흙=진흙, 불+바위=용암, 잎사귀+바람=씨앗 — 세계관 문서에 이미 예시 있음)
  - Tier 2: Tier1 + 원소, 혹은 3개 조합 (Fusion Field: 정해진 타일 위 3개 이상 조합 — '생명의 심장' 구역 기믹)
  - Tier 3+: Whisper 부여물이 재료로 들어가는 상위 조합
- **볼륨 목표 [결정 필요]**: MVP 기준 조합 결과물 40~60종, 정식 기준 레이어당 60~80종 제안. (Doodle God 1편이 ~115종, Little Alchemy가 ~500종)
- 실패 조합 처리: 재료 소모 없이 "반응 없음" + 힌트 게이지 누적 제안 (스트레스 최소화).

### 2.3 Build (건조)
- 오브젝트 여러 개를 붙인 뒤 부여를 통해 하나의 복합 오브젝트로 → 특정 특성 획득.
- Fusion이 "아이템 크래프팅"이라면 Build는 "구조물 크래프팅". 제0세계 구축의 핵심 도구.

### 2.4 Echo (복제)
- 한 번 융합한 소재는 솥단지에서 자동 복제 가능. 기본 재화 소모.
- 역할: **재채집 노가다 제거.** 발견은 한 번만 어렵고, 재사용은 싸게. 이게 있어야 Build/세계 구축이 노동이 아니라 놀이가 됨.

### 2.5 Whisper (부여) — 시그니처 시스템
- 만든 물체에 무언가를 "속삭여" 부여:

| 단계 | 부여 | 해금 | 효과 |
|---|---|---|---|
| 1차 | 에너지 | 1차 업적 | 오브젝트에 동력 부여 (기계 작동 등) |
| 2차 | 마력 | 2차 업적 | 마법적 특성 부여 |
| 3차 | 생명 | 3차 업적 | 생명 부여 (생물 창조) |

- Whisper 재화는 **특정 조건의 맵에서만 얻는 한정 재화** → 게임의 페이싱 조절 장치. 회차(Loop)마다 제한이 점진 해제됨.
- 에너지→마력→생명 3단계가 레이어 테마와 대응됨(기계=에너지, 마법=마력, 자연/신성=생명). 각 Whisper 유형의 재화를 대응 레이어에서 얻게 하면 탐험 동기가 명확해짐. **[제안]**
- 정화의 계곡의 "Whisper 이상 반응"(부정 피드백, 타일 뒤틀림)은 Balance 시스템의 첫 노출로 연결하면 좋음.

### 2.6 Balance (리스크 관리) [결정 필요 — 정의부터]
시스템 문서에도 "정의를 잘 해야 할 것 같음"이라 되어 있음. **제안 방향 3안**:
- A안 (오염 게이지): Whisper 남용 시 세계 오염 수치 상승 → 제0세계에 왜곡 발생. 단순, 구현 쉬움.
- B안 (조화 밸런스): 4원소/속성 간 균형 지표. 한쪽으로 치우친 세계는 레이어 1처럼 "정체"함 → 멸망 원인의 재현. 내러티브와 강하게 연결되지만 밸런싱 어려움.
- C안 (MVP 제외): Balance는 정식 버전으로 미루고 MVP에서는 Whisper 재화 한정만으로 리스크 표현.
- **추천: MVP는 C안, 정식은 B안 지향.** B안이 "선배들의 세계가 왜 멸망했는가"라는 스토리와 게임 시스템을 하나로 묶는 제일 예쁜 답이라서.

### 2.7 Loop (회차)
- 회차 반복으로 해금: 신기능, Whisper 재화 제한 해제, 멀티 엔딩.
- [결정 필요] 회차 시 유지되는 것(발견한 레시피 도감?)과 초기화되는 것(세계 상태?) 정의 필요. **제안: 레시피 도감·업적은 영구, 세계/인벤토리는 초기화** — 발견의 재미는 지키되 구축은 다시.

## 3. 콘텐츠 구조

### 3.1 제0세계 (Layer of Possibility) — 플레이어의 캔버스
- 빈 공간에서 시작, 채집한 원소와 Whisper로 세계를 구축.
- 타일 배치 → 오브젝트 배치 → Build 복합 구조물 → Whisper로 생명/동력 부여 순의 성장 곡선.
- [결정 필요] NPC 존재 여부, 아이템 상호작용 범위 (세계관 문서에 미결로 표기됨).
- [제안] 구축한 세계가 엔딩 판정의 대상이 되게: 무엇을 얼마나, 어떤 밸런스로 만들었는가.

### 3.2 제1세계 (Layer of Nature) — 첫 탐험 레이어, 5구역
문서 기준 정리:

| 구역 | 이름 | 역할 | 핵심 기믹 |
|---|---|---|---|
| 1 | 시작의 숲 | 튜토리얼, 기본 수집 | 이동/채집/조합/인벤토리 학습, 첫 조합(물+돌=진흙 등), 타일→VOID 학습 |
| 2 | 고요의 화원 | 자연×유적 비주얼 전달 | 비대칭 맵 시야 유도, 나비 추적 퍼즐, Whisper로 수정 드랍 |
| 3 | 생명의 심장 | 중앙 허브 | Whisper 튜토리얼, Fusion Field(3개 이상 조합), 4계절 방향 구역 |
| 4 | 푸른 절벽길 | 수직 이동, 퍼즐 | 움직이는 플랫폼 타이밍, 석탑 반응장치, 일방통행 길 |
| 5 | 정화의 계곡 | 첫 이상현상, 분위기 전환 | Whisper 이상 반응, 왜곡된 루프 길(카메라 트릭) |

- 1구역은 타일 8종(T0~T5B) / 오브젝트 15종(O0~O6) / 채집 아이템 9종 스펙이 이미 표로 확정 수준. **여기가 Build#1 (250809 목표로 문서에 표기됨) 범위.**
- 구현 난이도 경고 두 개 (문서에서 이미 물음표 달린 것들):
  - 물 반사 연출(2구역) — 도트에서 반쯤 잠긴 신전 반사는 비용 큼. 정적 스프라이트로 눈속임 추천.
  - 왜곡 루프 길(5구역) — "지도상 직진인데 실제 루프"는 아이소메트릭 타일맵에서 텔레포트 이음새로 구현 가능. 다만 폴리싱 비용 있음. MVP 제외 추천.

### 3.3 제2~5세계
- 컨셉만 존재. **정식 로드맵으로 미루고, MVP에서는 컷신/실루엣 티저로만 노출** 추천 (다음 세계가 있다는 기대감만).

## 6. MVP 스코프 제안 (Build#1)

**목표: "채집→조합→발견"의 재미가 성립하는가 검증**

포함:
- 제1세계 1구역 (시작의 숲) 전체 — 타일 8종, 오브젝트 15종, 아이템 9종
- Collect + Fusion (레시피 20~30종) + 인벤토리
- 솥단지 UI, 레시피 도감(발견 기록)
- 캐릭터 이동/채집 애니메이션, 낮/밤 사이클 (세계수 구역이 저녁~새벽 한정이므로 필요)
- 세이브/로드
- mac/Win 빌드 파이프라인 확립

제외 (정식으로):
- Whisper / Build / Echo / Balance (2구역 이후와 함께)
- 제0세계 구축 모드 (Fusion 검증 후에)
- NPC, 컷신, 2~5레이어

이유: 시스템 문서에도 "1구역에서는 Fusion에 집중"이라고 명시돼 있음. 그 방침 그대로 가는 게 맞다고 봄.

## 7. 결정 필요 리스트 (우선순위순)

1. **MVP 스코프 승인** — §6 안대로 갈지
2. **엔진 확정** — Godot 4 추천, 이견 있으면 지금
3. **타일 규격** — 아트 재작업 방지를 위해 최우선 확정

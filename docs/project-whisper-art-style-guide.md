# Project Whisper — 아트 스타일 가이드 v1.0

> 작성: Kana / 2026-07-05
> 상태: **확정안** (KOAL 위임: "아트 스타일은 카나가 잡고, 타일 규격은 일반적인 픽셀 isometric 규격")
> 기존 파이프라인 문서(Dreamina → 픽셀화 → 리터칭)와 호환되도록 설계.

---

## 1. 타일 규격 (확정)

**표준 2:1 아이소메트릭 다이아몬드.**

| 항목 | 값 | 비고 |
|---|---|---|
| 기본 타일 | **128 × 64 px** (다이아몬드, 2:1) | Godot TileMap `Isometric` + `Diamond Down` |
| 그리드 셀 | 128 × 64 | 타일 1칸 = 논리 좌표 1칸 |
| 벽/높이 단위 | 32 px (타일 높이의 절반) | 언덕/절벽 층 표현 시 |
| 소형 오브젝트 캔버스 | 128 × 128 | 꽃, 풀, 돌 등 — 기존 파이프라인의 128×128 출력과 일치 |
| 대형 오브젝트 캔버스 | 128 × 256 / 256 × 256 | 나무, 바위 군집. 세계수는 512 × 512 별도 |
| 캐릭터 캔버스 | 96 × 96 | 실 몸체 높이 ~56–64 px (타일 대비 1타일 반 느낌) |
| 최종 배포 해상도 | 1920 × 1080 기준, 정수 스케일 | Godot viewport stretch = `canvas_items`, snap 2D 활성 |

- **왜 64×32가 아니라 128×64인가**: 기존 아트 파이프라인이 128×128 리사이즈를 기준으로 이미 확정돼 있고, 레퍼런스(Eastward, Hyper Light Drifter)급 디테일은 64폭 타일로는 안 나옴. 128이면 AI 레퍼런스의 형태 정보를 덜 버리고 픽셀화할 수 있음.
- 오브젝트 기준점(origin)은 **바닥 다이아몬드의 중심**. Godot Y-sort가 이 기준으로 정렬됨. 스프라이트 제작 시 바닥 접지면을 캔버스 하단 중앙 다이아몬드에 맞출 것.

## 2. 스타일 정의

**"시간이 멈춘 세계의, 부드럽고 빛나는 도트."**

- 계열: Eastward의 밀도 + Sky/Journey의 빛 연출을 도트로. 레트로 8-bit 아님 (마스터 프롬프트의 "not 8-bit" 방침 유지).
- 형태: 둥글고 읽기 쉬운 실루엣, 볼드한 색 면. 과도한 노이즈 디테일 금지.
- **아웃라인: 순수 검정 금지.** 외곽선은 해당 면 색보다 2단계 어두운 동일 계열 색으로 1px (selout 방식). 내부 선은 최소화.
- **라이팅: 우상단 소프트 라이트 고정** (마스터 프롬프트와 동일). 모든 타일·오브젝트·캐릭터가 같은 방향. 스프라이트에 그림자 굽지 말 것 — 그림자는 엔진에서 타원 블롭(검정 40% 투명)으로 처리.
- 디더링: 원칙적으로 금지. 물 표면, 발광 그라데이션 등 큰 면에만 예외적 허용.
- 발광(세계수, 수정, Whisper 이펙트): 스프라이트에 굽지 않고 **엔진 추가 레이어**(additive blend + 약한 glow)로. 낮/밤 전환 시 발광만 살아나야 하므로 분리 필수.

## 3. 팔레트 (Layer 1: Nature 기준)

마스터 프롬프트의 "green / brown / blue / violet" 제한 팔레트를 실제 램프로 확정. **이 22색 밖의 색은 리터칭 단계에서 이 팔레트로 스냅할 것.**

```
── 초록 (풀/나무)      #1b3a2a  #2e5d3b  #4d8b4f  #7ab567  #a8d982
── 갈색 (흙/나무기둥)  #3a2a20  #5c4433  #8a6a4a  #b59268
── 파랑 (물)           #1e3a5c  #2e6b8a  #4aa3b8  #8fd4d9
── 보라 (신비/세계수)  #3a2a5c  #6b4a9e  #9e7ad9  #d9b8ff
── 중성 (돌/빛/UI)     #2a2a33  #6e6e7a  #b8b4a8  #e8dfc8  #faf5e6
```

- 시간대(낮/저녁/밤) 표현은 **팔레트를 늘리지 말고** 엔진에서 CanvasModulate/LUT 틴트로. 저녁~새벽 한정 구역(세계수 맵)은 base를 어둡게 + 보라 램프 발광 강조.
- Layer 2~5는 추후 계열별 파생 팔레트 (예: Science=청록+회색, Steampunk=구리+갈색). 구조는 동일하게 5램프 유지.

### 3.1 구역별 변주 규칙 (중요)

구역(N개)마다 분위기는 달라야 하지만, 구역 수만큼 팔레트를 만들면 아트 물량이 곱으로 터진다. **3층위로 분리:**

| 층위 | 무엇이 바뀌나 | 비용 |
|---|---|---|
| **레이어 공통** (전역) | 타일 규격, 아웃라인 방식, 라이팅 방향, 캐릭터 | 0 — 절대 불변 |
| **레이어별** | base 5램프 팔레트 (위 22색이 Layer 1의 것) | 레이어당 팔레트 1식 |
| **구역별** | ① 액센트 램프 1~2개 교체/추가 ② 엔진 틴트·조명 ③ 구역 고유 타일/오브젝트 소수 | 구역당 액센트 4~5색 + 고유 에셋 몇 개 |

**Layer 1 다섯 구역 적용안:**

| 구역 | base | 구역 액센트 | 엔진 연출 |
|---|---|---|---|
| 1 시작의 숲 | 공통 | — (base 그대로 = 기준점) | 새벽빛, 낮은 채도 틴트 |
| 2 고요의 화원 | 공통 | **적/분홍 램프** (#8a3a4a #c96a7a #f0a8b8) — 붉은 꽃·수정 | 오후 햇살, 따뜻한 틴트 |
| 3 생명의 심장 | 공통 | 4계절 액센트 (봄=분홍, 여름=진녹, 가을=주황 #d98a3a, 겨울=한랭청 #a8c8d9) | 방향별 지역 틴트 |
| 4 푸른 절벽길 | 공통 | 한랭 청회색 램프 (#2a3a4a #4a5c70 #8098ad) — 석탑·협곡 | 어두운 앰비언트 + 국소 광원 |
| 5 정화의 계곡 | 공통 | 왜곡 액센트: 채도 튄 시안/마젠타 소량 (#4ad9c8 #d94ab8) | RGB 오프셋 셰이더, 불규칙 발광 |

- 구역 고유 타일/오브젝트(예: 2구역 수정 꽃, 4구역 이동 플랫폼, 5구역 장치)는 base 팔레트 + 해당 구역 액센트만 사용.
- 원칙: **플레이어가 구역을 넘어갈 때 "색이 달라졌다"고 3초 안에 느끼되, 같은 세계라는 건 의심하지 않게.** base가 그걸 보장하고 액센트가 차이를 만든다.
- MVP(1구역)는 base 22색만으로 진행 — 액센트 확정은 각 구역 착수 시점에.

## 4. 캐릭터 (주인공)

- 컨셉: 갓 태어난 컨스트럭터. **소형 고양이형 실루엣** (세계관 문서의 고양이 교감 기믹, "제다이 고양이" 레퍼런스 반영).
- 규격: 96×96 캔버스, 몸체 ~56–64px. 실루엣이 타일 위에서 한눈에 읽혀야 함 — 머리:몸 비율 1:1 정도의 데포르메.
- 방향: 아이소메트릭 4방향 (SE / SW / NE / NW). 좌우는 미러 허용 → 실제 제작은 2방향 × 미러.
- 필수 애니메이션 (MVP):

| 동작 | 프레임 | fps |
|---|---|---|
| Idle | 4 | 6 |
| Walk | 6 | 10 |
| 채집 (Gather) | 6 | 10 |
| 놓기/배치 | 4 | 8 |

- 포인트 컬러: 몸은 중성 램프(크림/회색), **눈과 무늬에 보라 램프** — "속삭이는 자"의 아이덴티티 컬러가 보라.

## 5. 타일·오브젝트 제작 우선순위 (1구역 = Build#1)

기존 문서의 ID 체계 그대로 사용.

**Batch 1 — 게임이 서는 최소셋**: T2A 풀밭, T1 흙길, T5A 물, T0 VOID, O2A 나무, O6 돌, 캐릭터 Idle+Walk
**Batch 2 — 채집 루프 완성**: T2B/C/D, T4 진흙, T5B, O1A 꽃, O3A 바위, O4 흙언덕, O5A 풀, 채집 애니메이션
**Batch 3 — 분위기**: O1B~E, O2B~F, O3B, O5B (꾸밈 변형들), O0 세계수(512), 발광 이펙트

- VOID 타일(T0): "채집당해 비어버린 세계"의 상징. 단순 검정이 아니라 **가장자리가 보라로 미세하게 일렁이는 어두운 구멍**으로 — Whisper 세계관의 시각 시그니처로 쓴다.

## 6. Dreamina 마스터 프롬프트 개정안

기존 프롬프트 유지 + 팔레트 구체화만 추가:

> 기존 프롬프트 끝의 "limited fantasy palette (green, brown, blue, violet)" 를
> **"limited fantasy palette — moss green #4d8b4f, earth brown #8a6a4a, lake blue #4aa3b8, mystic violet #9e7ad9 as dominant tones"** 로 교체.

파이프라인 순서는 기존 PDF 그대로: Dreamina 생성(1:1) → 배경 제거(remove.bg) → 128 리사이즈 → 픽셀화 → **리터칭에서 §3 팔레트로 스냅 + §2 selout 아웃라인 정리**.

## 7. UI 방향 (MVP 최소)

- 인벤토리/솥단지 UI: 다크 베이스(#2a2a33) + 크림 텍스트(#faf5e6) + 보라 액센트. 프레임은 둥근 모서리의 단순 도트 패널, 장식 최소.
- 폰트: 한글 지원 도트 폰트 — **Galmuri** (SIL OFL, 무료) 추천. 11/14px 두 단계.
- 커서/선택 표시: 타일 위 다이아몬드 하이라이트 (보라, 펄스 애니메이션).

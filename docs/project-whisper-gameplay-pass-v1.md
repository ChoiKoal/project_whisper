# §5. 솥단지 전 레이어 배치 (저비용 선행)

## 5.1 현황 (코드 확인 결과 — 반드시 먼저 읽을 것)

**컨셉(오너)**: 방랑자가 각 세계에서 자기 솥을 불러낸다. 조합대는 세계마다 다른 정비대가 아니라 **같은 솥단지**여야 하고, 세계별 정체성은 **불꽃 색**으로만 준다.

코드 실측(2026-07-07):

| 레이어 | 조합대 스폰 | 실제 노드 | object_id | 발광 | 상호작용 경로 |
|---|---|---|---|---|---|
| 홈 섬 | `home_legend.json` 심볼 `C` | **`Cauldron` 클래스** (`cauldron.gd`) | `cauldron` | violet pool | `gatherable` 그룹 + `on_interact()` → FusionUI 자동 바인드 |
| L1 시작의 숲 | `map_legend.json` 심볼 `C` | **`Cauldron` 클래스** | `cauldron` | violet pool | 동일 (E 조합) |

| L3 클락워크 | `clockwork_city.gd::_spawn_workbench()` | **plain `Sprite2D`** `l3_workbench.png` | meta `workbench` | **orange** pool | — |

**L4/L5 조합대 부재 여부 → 결론: 부재 아님. L2~L5 전부 조합대가 스폰된다.** 세션이 `_loader.l2_workbench_special_cell()`(레전드 `special.workbench_cell`)에 정비대급 조합대를 스폰하고, 레이어별 색 발광 풀(cyan/orange/gold/gold)을 붙인다. 스폰 위치는 스폰 서편 2~3칸(§A-7 "first craft ≤4분" 페이싱).

**두 가지 실측 이슈 (이 스코프가 해결):**

1. **스킨 불일치 (오너 지적의 실체).** 홈/L1은 솥단지(`cauldron.png`), L2~L5는 레이어별 **정비대 스킨**(`l2~l5_workbench.png` — L3 brass, L4 amethyst, L5 ivory 등 제각각). 오너의 "정비대 스킨 제각각"이 코드에 그대로 있음.
2. **상호작용 경로 불일치 (숨은 정합성 리스크).** L2~L5 조합대는 `Cauldron` 클래스가 아니라 meta만 단 `Sprite2D`이고 **`gatherable` 그룹에도, `on_interact()`도 없다.** `InteractionController._do_interact()`는 `gatherable` 그룹 + `on_interact()` 대상에만 E 조합을 라우팅하고, `FusionUI._autobind_cauldrons()`는 `Cauldron` 클래스만 바인드한다. **즉 현재 L2~L5 조합대는 표준 E-조합 경로로는 열리지 않을 개연성이 크다.** (별도 오픈 경로가 확인되지 않음 — 구현 시 반드시 재현/확인.) 이 통일 작업이 그 리스크도 함께 닫는다.

## 5.2 설계 — 「어디서나 솥단지」

**단일 원칙: 조합대 노드는 전 레이어에서 동일한 `Cauldron` 클래스. 레이어 정체성은 불꽃(발광 풀) 색으로만.**

- **아트**: `l2~l5_workbench.png` 폐기(또는 보류). 전 레이어 `cauldron.png`/`cauldron_bubble.png` 재사용(홈/L1과 동일한 버블 애니 + breathing pulse). 방랑자가 "자기 솥을 불러낸다" 컨셉과 1:1.
- **불꽃 색(레이어 정체성)**: 기존 발광 풀 에셋을 그대로 재사용해 색만 배정 —
  - 홈/L1 = **보라**(`light_pool_violet.png`, 현행 유지)
  - L2 = **시안**(`light_pool_cyan.png`) — 정전된 문명
  - L3 = **주황**(`light_pool_orange.png`) — 마지막 온기
  - L4 = **금색**(`light_pool_gold.png`) — 아케인/봉인
  - L5 = **호박/앰버**(`light_pool_amber.png` 필요 시 신규 1종, 없으면 gold 재사용) — 신성
  - → §5 스펙상 "시안/주황/보라·금/호박" 그대로. 색 배정 표 확정.
- **스폰 위치**: 현행 `workbench_cell`(스폰 근처 고정) 유지. 위치 변경 없음 — **스폰 노드 타입만 교체**.
- **상호작용**: `Cauldron`으로 교체하면 `gatherable` 그룹 등록 + `on_interact()`가 딸려와 FusionUI 자동 바인드가 전 레이어에서 균일하게 성립(5.1 이슈 2 자동 해결).

## 5.3 구현 노트 (저비용 — 선행 가능)

- 세션 4곳(`terminal_station/clockwork_city/mage_tower/cathedral`)의 `_spawn_workbench()`를 **공통 헬퍼로 수렴**: `_spawn_cauldron(cell, flame_pool_path)`. 각 세션은 색 인자만 다르게 호출.

- **회귀 안전장치**: 홈/L1은 이미 `Cauldron`이라 변경 없음 → 표준 경로 회귀 위험 0. L2~L5는 노드 타입만 상향되므로 기존 세이브의 `l2_workbench_cell` 좌표와 호환.

# §1. NPC 퀘스트 시스템

## 1.1 컨셉 — 「살아있지 않은 잔재」

레이어의 주민은 **죽어 있다.** 하지만 완전히 사라지진 않았다 — 마지막 동작에 얼어붙은 잔재로 남아, 방랑자에게 말을 건다(스토리라인 §3 미니 아크 "규명": NPC와의 교류로 멸망 원인을 목격). 이들은 살아있지 않으므로 **대사는 짧고, 쓸쓸하고, 가끔 어긋나게 웃긴다**(자기가 죽은 걸 모르는 마법사 잔영 등). flavor 텍스트 톤의 연장이다.

**홈 섬 「메아리(Echo)」**는 예외적 상시 조력자다(스토리라인 §6): 솥단지에 깃든 목소리, 무언 주인공의 유일한 대변자. 튜토리얼 내레이션 + 상시 의뢰 발급자.

## 1.2 NPC 로스터 (레이어당 1~2 + 홈)

기존 각 레이어 진상 조각(마지막 로그 스크린/멈춘 로봇/마법사 잔영/석화 피조물/세계수)과 연계한다.

| NPC | 레이어 | 정체 | 톤 키워드 | 진상 연계 |
|---|---|---|---|---|
| **메아리 (Echo)** | 홈 섬 | 솥단지에 깃든 첫 컨스트럭터의 속삭임 잔향 | 다정하지만 어딘가 공허, 튜토리얼 | (프레임) 전 진상의 해설자 |
| **시들지 않는 노목** | L1 시작의 숲 | 죽었으나 잎을 떨구지 못하는 나무 | 느리고 늙고 온화 | 세계수(EG 진상) |
| **마지막 로그를 재생하는 안내 로봇** | L2 터미널 | 관제 안내 AI, 같은 안내를 무한 반복 | 사무적·고장난 명랑함 | 마지막 로그 스크린 |
| **태엽이 반쯤 남은 파수 로봇** | L3 클락워크 | 멈춰가는 경비 기계 | 끊기는 문장, 의무감 | 멈춘 로봇 |
| **자기가 죽은 걸 모르는 마법사 잔영** | L4 마탑 | 붕괴 순간 얼어붙은 잔상 | 오만·엉뚱, 코믹 최다 | 마법사 잔영 |
| **기도하다 굳은 석상** | L5 대성당 | 응답을 기다리다 석화된 신도 | 조용하고 경건·체념 | 석화된 피조물 |

**레이어당 1기 기본**(L2·L4는 서브 잔재 1기 추가 여지 — 문서상 후보로만, MVP는 6 NPC).

## 1.3 퀘스트 3유형 (각 NPC 4~6개 체인)

각 NPC 체인은 아래 3유형을 섞는다. 핵심은 **정답 아이템명을 직접 부르지 않는 것**(조합/배치에 추리·목적 부여).

**(A) 제작 의뢰 — 수수께끼형.** 정답 아이템명 대신 **속성/이미지를 묘사**한다. recipes.json의 `hint` 톤("물과 땅이 만나면", "축축한 곳의 초록")을 그대로 계승. 완료 판정 = `item_crafted`(target = 정답 아이템 id, `_target_matches`가 D↔I 별칭 해소).
  - 예(L1 노목): *"…목이 마르구나. 축축한 흙에서 돋는 걸, 한 줌."* → 정답 D07(습지 이끼류).

**(B) 배치 의뢰 — "내가 보이는 곳에".** 배치에 서사적 목적 부여. *"내가 보이는 곳에… 빛나는 걸 하나 놓아줘. 어두워서."* 완료 판정 = `placed_object_placed`(홈/그로브 배치 신호). MVP는 "아무 배치 1회"(target `any`)로 시작, 확장 시 좌표/아이템 제약.
  - 배치 보상으로 **전용 배치물**(NPC 곁에 남는 오브젝트)을 해금 → 세계에 흔적이 쌓이는 감각.

**(C) 회고 의뢰 — 진상 조각 연계.** NPC가 자기 죽음의 조각을 마주하게 한다. 완료 판정 = 진상 조각 조사(TruthShard `on_interact` → `GameState.collect_truth_shard`/`Codex.record_truth_log`). NPC 라인의 **마지막 퀘스트**로 배치 → 체인 클리어가 곧 그 세계의 "작별"(§3 미니 아크 4박자).
  - 예(L4 마법사 잔영): *"…내 실험 노트를 못 찾겠어. 저 잔영… 어, 저건 나잖아? 왜 저기 있지."* → 마법사 잔영 진상 조사.

## 1.5 구현 노트 — QuestManager 위 NPC 라인 추가

**현 구조(quest_manager.gd 실측):** L1~L5 **5개 독립 라인**이 공존한다. 각 라인은 자기 `active_id`/`progress` 포인터를 갖고, `_event()`가 5개 라인을 모두 tick하며, id 프리픽스(`L2-`~`L5-`)로 라인을 판별한다. 라인 head는 프리픽스로 로드 시 해소, `activate_lN_line()`이 첫 진입 시 dormant→활성. quests.json 스키마 = `{id, whisper, type, signal, target, count, next}`(type은 문서용, signal이 실제 훅).

**추가 방식 = 6번째 계열 「NPC 라인」 (프리픽스 `N-`).** 5중 라인 확장 패턴을 그대로 계승하되, **NPC 라인은 단선 체인이 아니라 NPC별 서브체인 다발**이라는 점만 다르다.

- **프리픽스 규칙**: `N-echo-Q1`, `N-oak-Q1`, `N-robot2-Q1` … `{N}-{npckey}-Q{n}`. 라인 판별은 `begins_with("N-")`.
- **다중 서브체인**: 기존 라인은 라인당 active 포인터 1개지만, NPC는 여러 NPC가 동시에 의뢰를 걸 수 있다. → `n_active: Dictionary`(npckey → active_id) + `n_progress: Dictionary`. `_event_line`의 line==N 분기에서 **활성 NPC 전부**를 순회하며 매칭 tick.
- **활성화 트리거**: 각 NPC와 첫 상호작용(신규 `QuestNPC` 노드, 아래) 시 `activate_npc_line(npckey)` 호출 → 해당 서브체인 head 활성. `activate_lN_line`과 동형(idempotent, done 검사).
- **보상 훅**: quests.json에 **`reward` 필드 신설**(`{recipes:[...], whisper:{attr,amt}, placeable:id, log:shard_id}`). `_complete(id)`에서 NPC 퀘스트면 `_grant_reward(q.reward)` 호출 → Codex/WhisperCurrency/배치 해금 디스패치. 기존 5라인은 reward 없음(하위호환).
- **영속화**: `to_dict`/`from_dict`에 `n_active`/`n_progress` Dict 추가(세이브 predates 시 빈 Dict). `reset()`에서 NPC 서브체인 전부 dormant.

**신규 월드 노드 `QuestNPC` (truth_shard.gd 패턴 재사용):**
- `Sprite2D` + `gatherable` 그룹 + `object_id`(예 `npc_oak`) + `on_interact()`. **TruthShard와 동일 패턴** — E 상호작용이 대화 카드 모달을 띄우고, 현재 활성 서브퀘스트의 `whisper`(의뢰문) + 완료 시 다음 의뢰문을 보여준다.
- 대화 카드 = TruthShard의 카드 모달(온디맨드 CanvasLayer)을 일반화한 `NpcDialogCard`(초상/이름/대사 3줄). 헤드리스 안전(카드 없이도 퀘스트 상태 진행).
- 세션 스폰: 각 레이어 세션(`_scatter_*`와 동렬)이 NPC 1기를 진상 조각 근처에 배치. 홈 세션은 메아리를 솥단지 곁에.

**HUD 연동**: 기존 `quest_started/progress/completed` 시그널 재사용 → 좌상단 상시 목표 + 도감 로그가 그대로 NPC 의뢰를 표시(추가 UI 최소). NPC 라인은 좌상단에 "현재 의뢰: [NPC명]"로 구분 렌더 여지.

## 1.6 전 퀘스트 텍스트 초안 (톤 가이드 + 대사)

**톤 가이드**: 1~2문장, 마침표보다 말줄임(…)이 잦다. 명령이 아니라 부탁/혼잣말. 죽은 존재의 어긋난 명랑함을 가끔 섞는다. 정답 아이템명 직접 호명 금지(속성 묘사). 무언 주인공이므로 NPC가 자문자답하듯.

### 메아리 (홈, 튜토리얼 + 상시) — `N-echo-*`
- **Q1(제작·튜토)**: *"솥이 비어 있어. …뭐든 두 개, 섞어봐. 태어나는 걸 보고 싶어."* → craft any / 보상: 레시피 힌트 1
- **Q2(배치·튜토)**: *"이 섬은 아직 네 게 아니야. 네가 만든 걸… 하나, 여기 놓아줘."* → place any / 보상: 전용 배치물(작은 등)
- **Q3(제작)**: *"차가운 밤이야. 오래 타는 걸 만들어 줄래. 흙과… 불씨 같은 걸."* → 정답 화롯불류 / 보상: Whisper 소량
- **Q4(회고)**: *"가끔… 내가 누구였는지 잊어. 저 물 위, 오래된 잎을 봐줄래."* → 세계수 진상 연계(홈 판) / 보상: 도감 기록

### L1 시들지 않는 노목 — `N-oak-*`
- **Q1(제작)**: *"…목이, 마르구나. 축축한 흙에서 돋는 초록을. 한 줌."* → D07 / 레시피 해금
- **Q2(배치)**: *"내 그늘이 보이는 곳에… 꽃을 하나. 오래 못 봤어, 색이란 걸."* → place / 전용 배치물(작은 꽃단)
- **Q3(제작)**: *"바람이 없어. 흔들릴 걸 만들어 줘. 가볍고, 마른 것."* → 정답 마른 갈대/씨앗류 / Whisper
- **Q4(회고)**: *"저기 물가에… 아주 어린 나무가 있지. 그게 나였을까. 봐줘."* → 세계수 진상 / 도감 기록

### L2 안내 로봇 — `N-robot-*`
- **Q1(제작)**: *"안내. 안내. …전력이, 필요합니다. 도는 것을, 죽은 것에서 만들어."* → 전지류 / 레시피
- **Q2(배치)**: *"승객 여러분. 제 앞에 등을 하나. 어두우면… 안내를, 못 합니다."* → place / 전용 배치물(안내등)
- **Q3(제작)**: *"복구. 복구. 회로를 이을… 가는 선 같은 걸 주세요. 감사합니다. 감사합니다."* → 전선/회로류 / Whisper
- **Q4(회고)**: *"마지막 방송을… 재생하시겠습니까? 예 / 예 / 예."* → 마지막 로그 스크린 진상 / 도감 기록

### L3 파수 로봇 — `N-guard-*`
- **Q1(제작)**: *"경비… 교대. 태엽이. 감을 것을. 돌아가는, 둥근 걸."* → 톱니/태엽류 / 레시피
- **Q2(배치)**: *"초소가, 보이는 곳에. 불을. 꺼지면… 나는 못, 본다."* → place / 전용 배치물(초소등)
- **Q3(제작)**: *"기름이. 굳었다. 미끄러운… 걸. 조금."* → 기름류 / Whisper
- **Q4(회고)**: *"내 앞의… 저 멈춘 동료를. 확인. 해줘. …나도, 곧."* → 멈춘 로봇 진상 / 도감 기록

### L4 마법사 잔영 — `N-mage-*` (코믹 최다)
- **Q1(제작)**: *"자네, 조수인가? 마침 잘 왔어. 빛나는 결정 하나 만들어 오게. 당장!"* → 마력 결정류 / 레시피
- **Q2(배치)**: *"내 연구실이… 어디 갔지? 아무튼 내가 보이는 데 등불 하나 놓게. 어두워서 실험을 못 해."* → place / 전용 배치물(마법등)
- **Q3(제작)**: *"봉인용 밀랍이 떨어졌네. 끈적하고… 잘 굳는 걸로. 빨리, 큰일 나기 전에."* → 봉인 밀랍류 / Whisper
- **Q4(회고)**: *"저 구석의 잔영… 나랑 똑 닮았군. 우스운 친구야. …잠깐. 왜 안 움직이지. 왜. 확인해 봐."* → 마법사 잔영 진상 / 도감 기록

### L5 석상 — `N-saint-*`
- **Q1(제작)**: *"…응답이, 없었습니다. 빛을 하나. 만들어 바쳐 주시겠어요."* → 성물/빛류 / 레시피
- **Q2(배치)**: *"제단이 보이는 곳에… 촛불을. 저는 손을, 움직일 수 없어서."* → place / 전용 배치물(봉헌 촛대)
- **Q3(제작)**: *"향이… 필요합니다. 타면서 피어오르는, 옅은 걸."* → 향/연기류 / Whisper
- **Q4(회고)**: *"저를… 봐주세요. 기도하다 굳은 저를. 응답은 오지 않았지만, 당신은 왔으니까."* → 석화된 피조물 진상 / 도감 기록

> 초안 = 6 NPC × 4 = 24 퀘스트. 확장 여지: NPC당 +1~2(제작/배치 반복형)로 4~6 체인 완성. L2/L4 서브 잔재 추가 시 +1 NPC 라인.

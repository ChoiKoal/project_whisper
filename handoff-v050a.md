### Built THIS phase (real elevation — Deliverable B)

**Sheet region → tile id map** is unchanged from the inherited slicers; the new work is height:

| Data / file | What it does |
|---|---|
| `data/map_height.txt` (NEW, 40×40) | Parallel height map. `0/1/2` = level, `/` = ramp. The central grass meadow (authored rows 17–23, the "풀 언덕") is +1; a compact core (rows 19–21, cols 10–22) is +2; 4 ramp cells (`(14,23) (15,23) (16,23) (18,17)`) are the walk-up crossings. Generated by `tools_gen_height.js`. VOID/water/gate cells stay height 0 — heights never float over the void. |
| `scripts/world/map_loader.gd` | Real elevation subsystem (replaces the old visual-only hill block). Loads the height file (`_load_height_data`), classifies it (`_classify_elevation` → `elevation`/`hill_cells`/`ramp_cells`), and builds it (`_build_elevation`). Also now honours the `layout/legend/height_path_override` exports (the home-island loader path — previously the overrides were declared but ignored). |

**Rendering** (`_build_elevation`): one **real `TileMapLayer` per level** (`Elev1` @ y=−32, `Elev2` @ y=−64),
same tileset, raised grass surface per hill cell mirroring the base variant. `collision_enabled=false`
on these layers (base `Ground` + the ledge body own all collision). Grass-topped **cliff faces** are
drawn on every downhill (screen-S / screen-E) transition via `_build_cliff_faces`, region-clipped to the
drop height and auto-varied by cell hash. Ramp cells get a worn-dirt slope (`_build_ramp_slopes`).

**Traversal / AStar** (`map_loader.can_traverse(a,b)` + `touch_controller.gd`): the TouchController was
switched from `AStarGrid2D` to a **height-aware `AStar2D`** — a point per walkable cell, an edge between
4-neighbours only when `can_traverse` allows (same level, or one side a ramp). Paths therefore never
route across a cliff; ramps are the only up/down link. Waypoints carry the cell's `height_offset` so the
walk visually climbs.

**Physics** (`_build_ledge_collision`): thin (78×10) rotated wall colliders lie along every non-ramp
diamond edge at the **un-lifted** grid position (the CharacterBody moves in flat logical space; the lift
is a sprite offset), so keyboard walking can't step off a ledge. Ramps have no ledge → the player walks
through them.

**Objects** (`_lift_hill_objects`): trees/flowers/rocks/scatter standing on a raised cell get the cell's
`height_offset` added to their Y, so they sit ON the plateau; Y-sort still reads their (now-lifted) world
Y, so cross-level sorting stays correct.
